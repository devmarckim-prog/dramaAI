## 4. 백엔드 핵심 코드 (Core Backend Logic)

### **A. Supabase 클라이언트 (`api/supabaseClient.js`)**
서비스 롤 키와 아논 키를 구분하여 클라이언트를 생성합니다.

```javascript
const { createClient } = require('@supabase/supabase-js');
const supabaseUrl = process.env.SUPABASE_URL;
const supabaseKey = process.env.SUPABASE_ANON_KEY;
const serviceKey = process.env.SUPABASE_SERVICE_ROLE_KEY;

const supabase = createClient(supabaseUrl, supabaseKey);
const serviceSupabase = createClient(supabaseUrl, serviceKey); // Admin bypass

module.exports = { supabase, serviceSupabase };
```

### **B. 인증 미들웨어 (`api/routes/api.js`)**
게스트 핑거프린트와 회원 JWT를 동시에 지원하는 하이브리드 인증 로직입니다.

```javascript
const authMiddleware = async (req, res, next) => {
  const authHeader = req.headers['authorization'];
  const guestFingerprint = req.headers['x-guest-fingerprint'];
  
  let token = null;
  if (authHeader && authHeader.startsWith('Bearer ')) {
    token = authHeader.split(' ')[1];
  }

  // 1. Token 인증 (회원)
  if (token && token !== 'null') {
    const { data: { user }, error } = await supabase.auth.getUser(token);
    if (!error && user) {
      req.user = user;
      return next();
    }
  }

  // 2. Fingerprint 인증 (게스트)
  if (guestFingerprint) {
    req.user = { id: guestFingerprint, isGuest: true };
    return next();
  }

  return res.status(401).json({ error: 'Unauthorized' });
};
```

## 5. AI 생성 엔진 (Edge Function Logic)

비동기로 7단계를 수행하는 Edge Function 로직의 핵심입니다 (`supabase/functions/generate/index.ts`).

1.  **Stage 1 (CORE)**: 제목/로그라인/줄거리 생성 (Haiku).
2.  **Stage 2 (OUTLINE)**: 전 회차 1줄 요약 생성.
3.  **Stage 3 (PLAN_DETAIL)**: 각 회차별 씬 리스트 생성 (Iterative Loop).
4.  **Stage 4 (SAMPLE)**: 1화 대본 미리보기 생성 (사용자 컨펌 단계).
5.  **Stage 5 (PRODUCTION)**: 캐스팅/장소/예산 산출 (Sonnet).
6.  **Stage 6 (PPL)**: PPL 기획안 생성.
7.  **Finalize**: 상태를 `done`으로 변경 후 종료.

## 7. 충돌 방지 및 재코딩 가이드 (Conflict Prevention)

다른 AI에게 코딩을 맡길 때 다음 요소를 **반드시** 강조하십시오.

1.  **ID 타입 일관성**: `projects.id`는 `TEXT` 타입으로 고정해야 합니다. (Numeric ID와 UUID, `g-` 접두사 게스트 ID가 혼재되기 때문)
2.  **JSON 자동 복구**: AI 응답이 끊기면 `try-catch`에서 문자열 끝에 `}`를 추가하여 파싱을 시도하는 복구 알고리즘을 소스 코드에 포함해야 합니다.
3.  **Polling 무결성**: 프론트엔드는 생성 중 서버를 신뢰하되, 로컬에서 `pct`를 부드럽게 올리는 `ProgressSmoother` 기능을 통해 UX를 개선해야 합니다.
