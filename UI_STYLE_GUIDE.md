# 🎨 DramaScript UI Style Guide & Development Rules

이 문서는 DramaScript 프로젝트의 시각적 일관성을 유지하고, 테마가 임의로 변경되거나 가독성이 떨어지는 문제를 방지하기 위한 공식 가이드라인입니다.

> [!IMPORTANT]
> **모든 개발 작업을 시작하기 전에 이 문서를 반드시 숙지하십시오.**

---

## 1. 테마 원칙 (Theme Principles)

### 🌟 사용자 페이지 (User-Facing Pages)
- **기본 테마**: **라이트 테마 (Light Theme)**
- **목적**: 시나리오 작업의 가독성을 높이고, 전문적인 문서 도구의 느낌을 주기 위함.
- **배경**: 밝은 색상 (`--paper`, `#f9f9f9`)
- **텍스트**: 짙은 색상 (`--ink`, `#121212`)

### 🛠️ 관리자 페이지 (Backend Office / Admin)
- **기본 테마**: **다크 테마 (Dark Theme)**
- **목적**: 운영 도구로서의 "Pro" 느낌과 일반 사용자 페이지와의 명확한 구분을 위함.
- **배경**: 어두운 색상 (`#0a0a0a`)
- **텍스트**: 흰색 또는 연한 회색 (`#ffffff`, `rgba(255,255,255,0.7)`)
- **버튼**: 전용 다크 버튼 또는 라이트 골드 스타일 사용

### 🏠 메인 페이지 가독성 (Main Page Visibility)
- **히어로 섹션 (Hero Section)**: 배경에 골드 그라데이션이 들어가더라도 메인 타이틀과 서브 타이틀은 반드시 고대비 색상(`--ink`, `--ink3`)을 사용해야 합니다.
- **배지 및 레이블**: "NO CREDIT CARD REQUIRED"와 같은 보조 텍스트에 투명도가 높은 흰색(`rgba(255,255,255,0.3)`)을 사용하는 것을 금지합니다.
- **버튼**: 메인 액션 버튼(`.btn-hero-primary`)은 배경색과 명확히 구분되는 골드 그라데이션과 그림자(`box-shadow`)를 사용하여 시각적 주목도를 높여야 합니다.

## 2. 색상 변수 사용법 (CSS Variables)

인라인 스타일(`style="color:#fff"`) 대신 반드시 아래의 CSS 변수를 사용해야 합니다.

| 변수 | 용도 | 라이트 테마 값 (Default) |
| :--- | :--- | :--- |
| `--paper` | 메인 배경색 | `#f9f9f9` |
| `--ink` | 메인 텍스트 색상 | `#121212` |
| `--ink3` | 보조 텍스트 (설명 등) | `#666666` |
| `--gold` | 포인트 색상 / 버튼 | `#d4af37` |
| `--border` | 경계선 / 카드 테두리 | `rgba(0,0,0,0.08)` |

### 💎 프리미엄 컴포넌트 클래스 (Premium Classes)
- **공통**: `.reveal` (스크롤 애니메이션), `.glass-morphism`
- **시스템**: `.debug-overlay-premium`, `.api-status-banner-premium`, `.notice-banner-success`
- **프로젝트 리스트**: `.project-slate-card`, `.project-card-new-premium`, `.pcg-badge-premium`
- **플랫폼 태그**: `.platform-tag-ott` (Red), `.platform-tag-tv` (Blue), `.platform-tag-web` (Green)

## 3. 개발 금지 사항 (Strict Prohibitions)

1.  **Hardcoded Colors in JS**: `projects_list.js` 등 JavaScript에서 HTML을 생성할 때 `style="color: #fff"`를 직접 넣지 마십시오. 테마가 바뀌어도 자동으로 대응되도록 클래스명만 부여하십시오.
2.  **Hardcoded Backgrounds in HTML**: `index.html` 본문에 `background: #000`과 같은 인라인 스타일을 쓰지 마십시오.
3.  **Theme Reversion**: 명시적인 사용자 요청 없이 라이트 테마를 다크 테마로 되돌리지 마십시오.

## 4. 가독성 체크 리스트 (Accessibility)

- [ ] 관리자 페이지(`section#page-admin`) 내부의 스타일은 전체 테마에 영향을 주지 않도록 스코핑(`Scoped`) 되어 있는가?
- [ ] **로컬 호환성**: `<base href="/">`가 제거되었으며, 모든 리소스가 상대 경로로 로드되는가?
- [ ] **캐시 방지 제거**: 로컬 실행 시 로딩 오류를 방지하기 위해 CSS/JS 링크에 `?v=...` 와 같은 쿼리 스트링이 없는가?

## 5. 운영 및 테스트 지침 (Operational & Testing Rules)

작업을 완료했다고 보고하기 전에 아래의 **실제 작동 여부**를 반드시 검증하십시오.

### 🧪 필수 테스트 시나리오
1.  **브라우저 확인 (Browser Validation)**: 
    *   `open_browser_url` 툴을 사용해 실제 화면을 띄워보고, 의도한 대로 디자인이 나오는지 육안으로 확인하십시오.
2.  **생성 기능 테스트 (Generation Flow)**:
    *   '새 프로젝트 만들기(Wizard)'에서 '생성 버튼'을 눌러보고, 실제로 생성 프로세스가 시작되는지 확인하십시오. 
    *   (기능적 결함이 테마 수정 중에 발생하지 않았는지 체크)
3.  **데이터 로딩 확인**:
    *   대시보드에서 기존 프로젝트들이 정상적으로 리스트업 되는지, 글자가 배경에 묻히지 않는지 확인하십시오.
4.  **콘솔 로그 체크**:
    *   수정 후 브라우저 콘솔에 붉은색 에러(RefError, SyntaxError 등)가 출력되지 않는지 확인하십시오.
5.  **메인 페이지 확인 (Main Page Check)**:
    *   로그아웃 상태 또는 메인 화면 진입 시 히어로 섹션의 모든 텍스트가 명확하게 읽히는지 확인하십시오. (흰색 배경에 흰 텍스트가 없는지 체크)

### 📝 보고 방식
- 모든 작업 완료 후에는 **`walkthrough.md`**를 작성하여, 어떤 부분을 테스트했고 어떤 결과가 나왔는지 사용자에게 보고하십시오. (가능하다면 스크린샷 포함)
- **가이드라인 업데이트 시 보고**: 사용자 피드백을 통해 본 `UI_STYLE_GUIDE.md` 파일이 업데이트될 경우, 결과값에 **"가이드라인에 다음 사항들을 업데이트했습니다: [업데이트 내용]"**이라고 반드시 명시하십시오.
- **최종 완료 테스트**: 작업을 마무리하고 "완료"라고 말하기 전, 반드시 실시간 환경에서 모든 클릭 반응과 전환이 정상적으로 이루어지는지 실제로 테스트하십시오. 테스트되지 않은 결과물은 미완성으로 간주합니다.

## 6. 필수 접속 및 실행 수칙 (Mandatory Access & Execution Rules)

이 프로젝트는 구글 OAuth 인증 및 AI 기능을 포함하고 있어, 단순 파일 열기 방식으로는 정상 작동하지 않습니다.

### 🚀 앱 접속 방법
1.  **절대 금지**: 브라우저에서 `index.html` 파일을 직접 더블 클릭하여 여는 방식 (**`file://` 주소**)은 사용하지 마십시오.
    *   **이유**: 브라우저 보안 정책(CORS)으로 인해 구글 로그인 및 외부 AI API 통신이 차단됩니다.
2.  **권장 방법**: 항상 로컬 서버 주소인 **`http://localhost:8081`**로 접속하십시오.

### 💻 서버 실행 방법
- 터미널에서 **`node serve.js`** 명령어를 입력하여 통합 서버를 실행하십시오.
- `static_server.js`는 단순 파일 서버이므로 AI 기능이 작동하지 않을 수 있습니다. 반드시 **`serve.js`**를 사용하십시오.

### ❓ 구글 로그인 실패 시 체크리스트
- [ ] 주소창이 `http://localhost:8081`로 시작하는가? (`file:///` 인 경우 즉시 주소 변경)
- [ ] 터미널에 `DramaScript AI Unified Server Running!` 메시지가 떠 있는가?
- [ ] 인터넷 연결이 안정적인가?
